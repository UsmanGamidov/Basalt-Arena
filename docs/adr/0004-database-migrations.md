# ADR 0004. Версионируемые миграции для прод-БД

- **Статус:** принято
- **Дата:** 2026-06

## Контекст

Локальная разработка ведётся на SQLite (нулевые внешние зависимости), прод — на
PostgreSQL (Supabase). Раньше схема катилась через `prisma db push` — без истории
и небезопасно для эволюции прод-данных.

## Решение

- Прод-схема (`schema.postgres.prisma`) управляется **Prisma Migrate**
  (`prisma/migrations/`). Деплой применяет `migrate deploy`.
- Деплой-скрипт `scripts/db-deploy.mjs` самобейзлайнящийся: на свежей БД применяет
  все миграции; на существующей БД из `db push` помечает `0_init` применённой и
  катит только новое; на уже-мигрированной — применяет новое.
- Новые миграции генерируются офлайн (`prisma:migrate:diff:postgres`) — локальный
  Postgres не требуется.
- Локальный dev остаётся на SQLite `db push` ради скорости и нулевых зависимостей.

## Последствия

- Безопасная, аудируемая эволюция прод-схемы; деплой не ломает существующую БД.
- Дуальность провайдеров: изменения схемы вносятся в обе схемы Prisma, миграция
  генерируется из postgres-схемы.
