# ADR 0001. Вести Architecture Decision Records

- **Статус:** принято
- **Дата:** 2026-06

## Контекст

Проект развивается несколькими людьми и нейросетями-ассистентами. Важные
архитектурные решения (выбор инфраструктуры, модели realtime, схемы БД) нужно
фиксировать, чтобы их можно было пересмотреть осознанно, а не «по памяти».

## Решение

Ведём короткие ADR в `docs/adr/` в формате «Контекст → Решение → Последствия».
Каждое значимое архитектурное изменение сопровождается новым ADR.

## Последствия

- Новый участник (человек или агент) понимает «почему так», а не только «как».
- Отклонения от типовых решений становятся явными и обоснованными.
