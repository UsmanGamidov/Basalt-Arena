# ADR 0002. Lean-инфраструктура без Redis и Docker

- **Статус:** принято
- **Дата:** 2026-06

## Контекст

Платформа деплоится одним сервисом на Render (API + статика клиента), БД —
управляемый PostgreSQL (Supabase). Нагрузка — один инстанс приложения.
Альтернатива — «тяжёлый» стек (Redis для сессий и распределённого rate-limit,
Docker/compose, горизонтальное масштабирование).

## Решение

Сознательно НЕ вводим Redis и Docker:

- **Сессии/ревокация** — `jti` access-токена в таблице `AuthToken` (БД), refresh —
  opaque UUID в БД. На одном инстансе этого достаточно.
- **Rate-limit** — `@nestjs/throttler` (in-memory) глобально + точечные лимиты на auth.
- **Realtime** — SSE без брокера (см. ADR 0003).

## Последствия

- Меньше внешних зависимостей и точек отказа, дешевле и проще эксплуатация.
- При переходе на несколько инстансов потребуется общий стор для сессий,
  rate-limit и шины событий (Redis) — это явная точка пересмотра данного ADR.
